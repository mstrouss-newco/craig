For a nonprofit board, built for AI to maintain

The calm chief of staff for your board.

craig keeps a foundation board running between meetings: schedule the next meeting, collect grant votes, gather signatures on resolutions, share documents, and keep the roster straight. Staff work from a dashboard, directors get a simple portal, and both read and write the same live record.

What craig does

Seven jobs, one place. Everything a small board needs to stay in good order, with nothing it does not.

☉

Meetings

Schedule meetings, track attendance, and watch a live quorum bar so you always know if you can do business.

✓

Grants

Submit a grant, collect each director's vote, and see who has responded. A grant approves itself the moment votes reach quorum.

✎

Signatures

Collect sign offs on resolutions and consents, with a clear record of who signed and when, including unanimous consents.

☷

Documents

Share decks, minutes, and policies by link, optionally tied to the meeting they belong to.

✉

Updates

Post short notes to the board between meetings so nobody is surprised at the next one.

☉

Governance

Keep the board roster, roles, terms, status, and the organization's name all in one tidy place.

How it is built

craig is deliberately simple. Each page is one self contained HTML file: the markup, the styling, and the JavaScript all live together. There is no framework, no build step, and no server to run.

Data lives in a hosted database and is read and written directly over a REST API, so a change a director makes is saved and visible everywhere at once. The whole thing is served as a static site on GitHub Pages, which means pushing to the repository is the only deploy step.

It is built this way on purpose: files you can host anywhere, with zero tooling, that keep working without anyone babysitting them.
