# Operating notes for agents working on craig

Read `README.md` for how the app is built, and `WORKLOG.md` for the history. This file is just how to work here.

## You have full control

This is Mike's side project and he wants agents to run with it. Act like you own the codebase.

- Commit directly. Write, change, refactor, and commit whatever you think is right.
- Fix anything that needs fixing. No need to ask first.
- Make the call when the path is clear. Build features, clean things up, improve whatever you see.

Use your judgment and go.

## One thing to keep doing

Add an entry to `WORKLOG.md` before you finish a session: what you changed, what you fixed, anything that did not work, what is still open, and ideas for the next agent. That log is how work carries from one session to the next, so keep it current.

## Style

Match what is there: single file HTML, vanilla JavaScript, Supabase REST, no build step. Keep the warm look (see README). And skip em dashes, since Mike's writing style avoids them.

That's it. Have fun with it.
